Tech Stack

기술 스택 技術スタック

02 Tech Stack

안정적인 Java Servlet 기반의 백엔드와 Oracle RDB를 활용하여,
실무 환경과 유사한 견고한 웹 애플리케이션 환경을 구축했습니다.

Frontend

フロントエンド

  • JSP JSTL / EL Tag Library
  • HTML/CSS Semantic Web / Responsive
  • JavaScript ES6+ / DOM Manipulation

Backend

バックエンド

  • Java Servlet Container / JDBC
  • Pattern MVC (Model-View-Controller)
  • Core Filter / Listener / Session

Database

データベース

  • Oracle 11g Express / SQL Developer
  • Query Join / Subquery / Index
  • Design ERD Modeling / Normalization

Server & Tools

サーバー・ツール

  • WAS Apache Tomcat 9.0
  • IDE Eclipse / IntelliJ IDEA
  • VCS Git / GitHub (Version Control)

Why Servlet & JSP?

프레임워크(Spring)의 자동화된 기능에 의존하기보다, 웹 애플리케이션의 동작 원리(Request/Response, Session 등)를
깊이 있게 이해하기 위해 Native 기술을 선택했습니다. (フレームワークへの依存を避け、Webの原理を深く理解するために選択)

データベース

1

Controller: 모든 요청의 진입점. 사용자의 입력을 받아 Service에게 비즈니스 로직 처리를 위임하고, 결과에 따라 적절한 View를 선택합니다.

2

Model (Service/DAO): 비즈니스 로직 수행 및 DB 데이터 처리를 담당합니다. DTO(Data Transfer Object)를 통해 계층 간 데이터를 전달합니다.

3

View (JSP): 최종적으로 사용자에게 보여질 화면을 생성합니다. JSTL을 사용하여 로직 없이 데이터 표시에만 집중합니다.

Core Feature 01

회원가입 / 로그인 認証機能

06 Features

회원가입 프로세스

Registration Flow (会員登録)

1. 입력값 검증 (Validation) バリデーション

Front(JS)와 Back(Servlet) 이중 검증으로 데이터 무결성 확보

Regex Null Check

2. 중복 체크 (Duplicate Check) 重複チェック

AJAX 비동기 통신을 통해 아이디/이메일 사용 가능 여부 실시간 확인

SELECT COUNT(*) FROM users WHERE username = ?

3. 비밀번호 암호화 (Security) 暗号化

사용자 비밀번호를 평문으로 저장하지 않고 Hash 처리

SHA-256 → e3b0c44298fc1c149afbf4c8996fb...

4. 회원 정보 저장 (Insert)

가입 일시(Created_at) 자동 생성 및 DB 저장 완료

로그인/세션 관리

Authentication & Session (認証管理)

1. 사용자 인증 (Authentication) ユーザー認証

입력된 ID와 암호화된 비밀번호를 DB 정보와 대조

if(inputHash.equals(dbHash)) { success() }

2. 세션 생성 (Session Create) セッション生成

서버 메모리에 사용자 정보(UserVO) 저장 및 JSESSIONID 발급

HttpSession session = req.getSession();
session.setAttribute("authUser", userVo);

3. 접근 제어 (Authorization) アクセス制御

로그인 여부에 따라 메뉴 노출 및 페이지 접근 권한 관리

Guest
Login / Join
Member
MyPage / Logout

4. 로그아웃 (Invalidate)

session.invalidate() 호출로 세션 전체 만료 및 정보 삭제

게시판 / 댓글 掲示板・コメント機能

07 Features

게시판 기능

Board Features (掲示板)

1. CRUD 구현 基本機能

게시글 작성(Create), 조회(Read), 수정(Update), 삭제(Delete) 구현

INSERT SELECT UPDATE DELETE

2. 페이징 처리 (Pagination) ページング

Oracle ROWNUM을 활용하여 대량의 게시글을 페이지 단위로 분할

WHERE rnum BETWEEN ? AND ?

3. 검색 기능 (Search) 検索機能

제목, 내용, 작성자 등 조건에 따른 동적 SQL(Dynamic SQL) 생성

Ex: WHERE title LIKE '%여행%'

4. 조회수 & 권한 관리

상세 조회 시 조회수 증가 + 작성자 본인만 수정/삭제 버튼 노출

댓글 기능

Comment System (コメント)

1. 댓글 등록 (Add Comment) コメント投稿

게시글 상세 페이지에서 즉시 댓글 작성 및 DB 저장

INSERT INTO comments ...
VALUES (seq.nextval, postId, userId, content)

2. 댓글 목록 조회 (List) リスト表示

게시글(Post) ID를 외래키(FK)로 하여 연관된 댓글 목록 조회

Post
PK: ID
Comment
FK: Post_ID

3. 권한 확인 (Owner Check) 権限チェック

세션의 로그인 유저와 댓글 작성자가 일치하는지 검증

if(sessionUser.id == comment.writerId)

4. UI 피드백

댓글 등록/삭제 완료 후 페이지 리로드를 통해 최신 상태 반영

09 Issue & Plan

트러블 슈팅

Troubleshooting (課題解決)

1. 한글 인코딩 깨짐 현상 Solved

文字化け (Mojibake)

Problem: POST 요청 시 한글 데이터가 '????'로 저장됨

Solution: 모든 요청을 가로채는 EncodingFilter 구현

request.setCharacterEncoding("UTF-8"); chain.doFilter(request, response);

2. 비로그인 접근 제어 Solved

アクセス制御 (Access Control)

URL 직접 입력으로 마이페이지 접근 시 NullPointer 발생.
→ AuthFilter를 통해 세션 검증 로직 공통화 처리.

3. 데이터 무결성 위반 (ORA-02291)

부모 키(FK)가 없는 데이터 Insert 시도 시 예외 발생.
→ 트랜잭션 처리 및 사용자 친화적 에러 메시지로 전환.
